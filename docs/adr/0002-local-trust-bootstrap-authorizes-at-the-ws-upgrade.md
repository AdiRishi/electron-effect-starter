# Local Trust Bootstrap, Authorized At The WS Upgrade

The shell mints one random bootstrap token per process and hands it to the spawned server **over inherited file descriptor 3**, not argv or env. This is deliberate, not exotic plumbing: argv and environment blocks are readable by other local processes on many platforms, while an inherited fd exists only in the child. The spawn config also _clears_ the `APP_BOOTSTRAP_TOKEN`/`APP_SERVER_PORT` env fallbacks so a developer's shell cannot silently override the envelope. Clients (the renderer via the bridge, or a plain browser via `POST /api/auth/bootstrap/bearer`) exchange that token for an opaque bearer session, and authorization is checked **once, at the `/ws` upgrade**.

The bearer itself never rides in a URL. Browsers cannot set headers on a WebSocket upgrade, so *some* credential must go in the query string — and URLs leak (proxy logs, browser history, `Referer`). The client therefore exchanges the bearer (over `Authorization`, where headers work: `POST /api/auth/ws-ticket`) for a **five-minute ticket**, and only that ticket is accepted in `/ws?wsTicket=`; a leaked socket URL exposes minutes of access, not the session. T3 Code does the same with stateless HMAC-signed tickets bound to persisted session rows; with this starter's in-memory process-lifetime sessions, an opaque random ticket in a pruned map keeps the identical exposure bound with far less machinery.

There is deliberately no per-RPC authorization. T3 Code checks scopes inside individual RPC handlers; this starter has exactly one trust level — "paired with the local server" — so per-handler checks would be ceremony. The RPC contracts still declare `EnvironmentAuthorizationError` in their error channels: that is not dead weight but forward-compatibility, so clients are already typed for the day per-RPC authorization is added. Likewise the bearer store is in-memory with no expiry and plain string comparison — proportionate to a loopback-only server with process-lifetime sessions, and the same posture T3 Code takes for its local bootstrap token.
